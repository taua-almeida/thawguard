{{ end }} {{/* components/decisions-approval-steps — the thaw policy explainer, shared by the aside's collapsed and expanded variants. Copy stays honest about what approval does: live forge re-check, head-SHA scope, one status context. */}} {{ define "components/decisions-approval-steps" }}
  1. 1
    Live head fetch

    Approval fetches the pull request's current head commit from the forge and re-syncs open pull requests; the eligibility preview is only the webhook cache.

  2. 2
    Head-commit-scoped exception

    The exception is recorded for that exact commit and never expires on its own; any new push invalidates it. If other open pull requests share the commit, approval pauses for an explicit all-PRs confirmation.

  3. 3
    One status context

    Thawguard recomputes and publishes only the {{ .RequiredContext }} commit status; merge blocking still depends on the forge requiring that context.
